How-to Geek正在寻找Linux编写者

您是一个熟悉Linux终端的有经验的作家吗？我们正在寻找像您一样的人在How-to Geek上编写Linux教程。

我们要找的是什么

我们正在寻找熟练使用Linux命令行的有经验的作家。理想的候选人对巴什了如指掌，能够向普通人解释这些极客的事情。

我们不是在寻找报道Linux新闻的作家，也不是在寻找只有图形Linux桌面经验的人。你可能会也可能不会写一些关于Linux桌面软件的Linux新闻和教程，但我们正在寻找一个有严肃的Linux终端经验的人。如果你有树莓派的经验，那也会更好。

这是一个自由撰稿人的职位，您将负责撰写分配给您的主题，但您也可以介绍我们尚未介绍的有趣的Linux终端文章。

以下是我们总是在新作家身上寻找的东西：

你本质上一定是个极客，总是希望学习更多的技术知识，让你的小玩意儿工作得更好。 你必须能够以一种清晰易懂的方式解释复杂的主题，即使对非专家也是如此。 你必须有创造力，并有能力产生文章的想法，采纳建议，并使主题有趣和令人兴奋。 您必须年满18岁，并且拥有自己的计算机。 你必须有扎实的英语写作能力。遗憾的是，我们甚至不得不提那件事。 你应该有一些基本的屏幕截图和图像编辑印章。具备HTML技能者优先。

为了让你对我们的期望有一个概念，这里有几个我们正在寻找的工作类型的例子：

Add a User to a Group(或Second Group on Linux)-这篇文章解释了组，并准确地分析了如何执行常见任务，并附有屏幕截图。 How to Compress and Extract Files Using the tar Command on Linux)-这篇文章解释了如何完成一项常见的任务，但更重要的是，它提供了示例，并实际解释了tar命令的各种开关的作用。 初学者极客：如何开始使用Linux终端-并不是一切都是基本的如何使用。这是一本试图使Linux终端变得易于理解的指南，也是您能够处理的更深入内容的一个示例。

解释为什么你的极客技能值得每月向数百万读者兜售。 你的名字和地址。 您以前在写作和/或博客方面的任何经验，特别是与Linux相关的经验。 不管你目前是否有工作，如果你有工作，你会做什么。 简要概述您熟悉的任何其他主题，以及您有权访问的操作系统、计算机和设备。 最重要的是：“我们想要一份写作样本。”如果您有以前的文章要展示，特别是您引以为豪的相关Linux作品，请在您的电子邮件中包含一个指向它的链接。如果你有个人博客、论坛账号或来自任何地方的评论者账号，也可以随意添加。
