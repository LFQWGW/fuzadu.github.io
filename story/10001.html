Ubuntu 16.04让Ubuntu再次激动人心

在过去的几年里，Ubuntu在Linux用户中的口碑并不是最好的--有些人甚至称它“无聊”。“如果你一直在犹豫要不要试一试，那就保住你的座位吧--Ubuntu 16.04”Xenial Xerus“不仅是一个激动人心的版本，而且有可能成为Linux生态系统的游戏规则改变者。

Ubuntu在2004年首次跳入Linux世界，并随之而来，彻底改变了Linux的面貌，将其从“只有经验丰富的极客才能使用”的时代带入了“人类的Linux”时代。现在，12年后的今天，他们可能即将重蹈覆辙，将它从一个全新的小项目变成最受欢迎的Linux发行版。Ubuntu16.04于今天发布，随之而来的是整个发行版中大量的改进。有许多更改可以改善最终用户的可用性和体验，还有一些潜在的里程碑式的更改可能会激起即使是最持怀疑态度的开发人员的兴趣。

Unity启动器可以移动到屏幕底部

相关：如何在Ubuntu 16.04上将Unity Desktop的启动程序移动到屏幕底部

多亏了Ubuntu麒麟团队，用户现在可以将Unity Launcher附加到屏幕底部，而不是被迫始终将其放在左侧。信不信由你，获得这一基本功能花了将近6年的时间。

有几种方法可以做到这一点，但最简单的方法是通过终端设备中的一个命令(尽管无可否认，这是一个相当长的命令)。使用Ctrl+Alt+T或从Dash打开您的终端，然后运行以下命令：

gsettings set com.canonical.Unity.Launcher launcher-position Bottom

如果您稍后决定不喜欢它，也可以通过运行以下命令恢复到左侧：

gsettings set com.canonical.Unity.Launcher launcher-position Left

这就是一切所需要的。

默认情况下，在线破折号结果处于关闭状态，并更新为“apt”命令

几年来，关于Ubuntu的Dash的在线搜索结果一直存在相当多的争议。有些人甚至(不准确地)称它们为“间谍软件”。Ubuntu16.04通过默认禁用搜索结果结束了这一争议。

GNOME软件取代Ubuntu软件中心

Ubuntu软件中心是Ubuntu名字上的另一个污点。它速度慢，不可靠，整体用户体验欠缺。Ubuntu 16.04通过将Ubuntu软件中心替换为GNOME的软件解决方案来解决此问题。Ubuntu采用GNOME软件是Canonical更多社区参与的一个很好的标志，如果总体上更好的话，他们愿意加入一个替代的软件。

同样，Canonical在Ubuntu 16.04中采用了一个全新的日历应用程序-这只是他们采用GNOME项目中更好的软件的另一种方式。

如果您更喜欢终端，那么10.04还为“apt”命令添加了新功能，因此您可以比以前更进一步地简化命令行包管理。Ubuntu16.04新增了apt autoremove，它取代了apt-get autoremove和apt purge包，取代了apt-get清除包。

Unity 7.4是迄今为止最流畅的Unity体验

我测试Ubuntu16.04和Unity 7.4已经有很长一段时间了，我不得不说，Unity 7.4是到目前为止我体验过的最流畅、最好的Unity体验。我支持12.04的基于Qt的Unity，但我很高兴看到Ubuntu16.04采用了它最好的特性。以下是Unity 7.4中最值得注意的更改：

Unity Dash中的会话管理快捷方式，如重新启动、关闭等。 加载应用程序时，图标出现在启动器中。 能够将Unity启动器移动到屏幕底部。 默认情况下禁用在线短划线结果。 应用程序菜单现在可以设置为‘Always Show’(始终显示)。 Unity Dash中的新滚动条。 外部存储/回收站现在显示打开的窗口数量。 添加到工作空间切换器的快速列表(Jumplist)。 能够在Unity快速列表中格式化驱动器(非常节省时间，但要小心)。 现在可以使用Alt+{num}打开与Logo+{num}类似的外部存储项目以打开应用程序。 Ubuntu主题改进了对客户端装饰的支持。

那可是很多好东西啊。

缺省情况下，Ubuntu 16.04支持ZFS

ZFS是一种非常流行的文件系统，因为它具有处理大数据集的可靠性，并且多年来一直是Linux社区非常热门的话题。Canonical已经决定ZFS支持是必要的，所以Ubuntu 16.04在默认情况下增加了对ZFS的支持。但是，默认情况下不启用ZFS，这是有意为之。由于大多数用户不需要ZFS，因此它最适合大规模部署。因此，虽然这很酷，但它不会影响大多数人。

Ubuntu Snappy有可能改变Linux的面貌

最后，Ubuntu 16.04将Ubuntu Snappy引入桌面，这是一个全新的包管理解决方案，有可能改变Linux的面貌。

基于Linux的操作系统有许多不同类型的版本结构，但最常见的两种是固定版本(又名稳定版本)和滚动版本。这两种常见的结构都有优缺点：固定版本为您提供了坚如磐石的基础系统，但通常使用的是过时的应用程序，必须用PPA之类的东西来补充。每当发布新版本以及所有最新的错误时，滚动发布都会让您尽快更新软件。Ubuntu Snappy是一种新的发布结构，它将两个系统的所有优点结合在一起。

可以将Snappy视为.deb文件和PPA的替代方案。这是一种新的应用程序分发形式，开发者可以在他们的应用程序准备好后立即以“快照”的形式向你发送最新版本的应用程序。对于开发人员来说，推出PPA要容易得多，速度也更快，而且如果Ubuntu的默认存储库包中没有包含PPA，那么您--用户--就不必去寻找PPA了。而且，如果某个版本有bug，很容易就能回滚到上一个稳定的版本。

此外，Snap的安装方式与您习惯的传统.deb文件不同。Snap安装为基于“只读”可挂载映像的应用程序，这意味着您不必担心应用程序是否打包为Ubuntu 16.04、16.10或任何其他版本-该Snap将在任何支持Snap的Ubuntu版本上运行。

桌面上的Snappy还处于早期阶段，所以你不会完全使用Ubuntu16.04来切换到快照。但基础已经奠定，随着时间的推移，抓拍应该会开始变得更加常见。事实上，Ubuntu将在未来发布一个类似于“Snap Store”的应用程序，很可能使用GNOME软件，使使用Snappy发现和安装应用程序变得更容易。
