如何使用命令行在Linux中查找文件和文件夹

大多数人会使用图形文件管理器来查找Linux中的文件，比如Gnome中的Nautilus、KDE中的Dolphin和Xfce中的Thunar。但是，无论您使用哪种桌面管理器，都可以通过多种方式在Linux中使用命令行查找文件。

使用查找命令

“find”命令允许您搜索您知道其大致文件名的文件。该命令的最简单形式是在当前目录中搜索文件，并递归搜索与提供的搜索条件匹配的子目录。您可以按名称、所有者、组、类型、权限、日期和其他条件搜索文件。

在提示符下键入以下命令将列出在当前目录中找到的所有文件。

find .

“find”后的圆点表示当前目录。

要查找与特定模式匹配的文件，请使用-name参数。您可以使用文件名元字符(如*)，但您应该在每个元字符前面加上转义字符(\)，或者用引号将它们引起来。

例如，如果我们要在Documents目录中查找所有以“pro”开头的文件，我们可以使用CD Documents/命令切换到Documents目录，然后键入以下命令：

find . -name pro\*

将列出当前目录中以“PRO”开头的所有文件。

注意：默认情况下，find命令区分大小写。如果希望搜索词或短语不区分大小写，请在find命令中使用-iname选项。它是-name命令的不区分大小写版本。

如果查找没有找到任何与您的条件匹配的文件，则不会生成任何输出。

find命令有许多选项可用于优化搜索。有关find命令的详细信息，请在终端窗口中运行man find并按Enter键。

使用LOCATE命令

LOCATE命令比find命令更快，因为它使用以前构建的数据库，而find命令在实际系统中搜索所有实际的目录和文件。LOCATE命令返回包含指定字符组的所有路径名列表。

数据库从cron定期更新，但您也可以随时自行更新，以便获得最新的结果。要执行此操作，请在提示符下键入以下命令：

sudo updatedb

在系统提示时输入您的密码。

定位命令的基本形式是查找文件系统上的所有文件(从根开始)，这些文件包含全部或部分搜索条件。

locate mydata

例如，上面的命令发现两个包含“mydata”的文件和一个包含“data”的文件。

如果要查找恰好包含且仅包含搜索条件的所有文件或目录，请将-b选项与LOCATE命令一起使用，如下所示。

locate -b ‘\mydata’

上面命令中的反斜杠是一个全局字符，它提供了一种将非特定文件名中的通配符扩展为一组特定文件名的方法。通配符是计算表达式时可以由一个或多个字符替换的符号。最常见的通配符是问号(？)和星号(*)，问号(？)表示单个字符，星号(*)表示连续字符串。在上面的示例中，反斜杠禁用了“*mydata*”对“mydata”的隐式替换，因此您最终只能得到包含“mydata”的结果。

mlocate命令是Locate新实现。它为整个文件系统编制索引，但搜索结果仅包括当前用户有权访问的文件。当您更新mlocate数据库时，它会在数据库中保留时间戳信息。这样，mlocate就可以知道目录的内容是否发生了更改，而无需再次读取内容，并且可以更快地更新数据库，并降低对硬盘的要求。

安装mlocate时，/usr/bin/locate二进制文件将更改为指向mlocate。要安装mlocate，如果您的Linux发行版中尚未包含它，请在提示符下键入以下命令。

sudo apt-get install mlocate

注意：我们将在本文后面向您展示一个命令，该命令允许您确定命令的可执行文件(如果存在)所在的位置。

MLOCATE命令使用的数据库文件与标准LOCATE命令使用的数据库文件不同。因此，您可能希望通过在提示符下键入以下命令来手动创建数据库：

sudo /etc/cron.daily/mlocate

在手动创建数据库或从cron运行脚本之前，mlocate命令不起作用。

有关LOCATE或mLOCATE命令的详细信息，请在终端窗口中键入man locate或man mlocate命令，然后按Enter键。两个命令显示相同的帮助屏幕。

使用WHERE命令

“Which”命令返回发出命令时调用的可执行文件的绝对路径。这在查找可执行文件的位置时很有用，该文件用于在桌面、面板或桌面管理器中的其他位置创建指向程序的快捷方式。例如，键入Firefox将显示的结果的命令，如下图所示。

默认情况下，Which命令仅显示第一个匹配的可执行文件。要显示所有匹配的可执行文件，请在命令中使用-a选项：

which -a firefox

有关Which命令的详细信息，请在终端窗口的命令提示符下键入“manWhich”(不带引号)，然后按Enter键。

使用Where is命令

Where is命令用于找出命令的二进制文件、源文件和手册页文件的位置。例如，在提示符下键入Where is Firefox：将显示结果，如下图所示。

如果您只想显示可执行文件的路径，而不想显示源和手册页的路径，请使用-b选项。例如，命令Where is-b Firefox将只显示/usr/bin/Firefox作为结果。这很方便，因为与搜索该程序的源代码和手册页相比，搜索程序的可执行文件的频率更高。您还可以仅搜索源文件(-s)或仅搜索手册页(-m)。

有关Where is命令的详细信息，请在终端窗口中键入man Where is，然后按Enter键。

了解WHERIS命令和WHERE命令之间的区别

Where is命令显示命令的二进制、源代码和手册页的位置，而Which命令仅显示命令的二进制文件的位置。

WHERIS命令在特定目录列表中搜索二进制文件、源文件和MAN文件，而WHERIS命令搜索当前用户的PATH环境变量中列出的目录。对于Where is命令，可以在该命令手册页的FILES部分找到特定目录的列表。

当谈到默认显示的结果时，Where is命令显示它找到的所有内容，而Which命令只显示它找到的第一个可执行文件。您可以使用前面讨论过的-a选项为Which命令更改这一点。

因为Where is命令只使用硬编码到该命令中的路径，所以您可能并不总是找到您要查找的内容。如果要搜索您认为可能安装在Where is命令手册页中未列出的目录中的程序，则可能需要使用带有-a选项的Which命令来查找该命令在整个系统中的所有匹配项。
