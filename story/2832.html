在Linux上将用户添加到组(或第二个组)

更改用户关联到的组是一项相当简单的任务，但并不是每个人都知道命令，尤其是将用户添加到辅助组时。我们将为您演练所有的场景。

在Linux上，可以将用户帐户分配给一个或多个组。您可以按组配置文件权限和其他权限。例如，在Ubuntu上，只有sudo组中的用户才能使用sudo命令获得提升的权限。

添加新组

相关：Linux中的SUDO和SU有什么不同？

如果要在系统上创建新组，请在命令后面使用groupadd命令，将new_group替换为要创建的组的名称。您还需要在此命令中使用sudo(或者，在不使用sudo的Linux发行版上，在运行该命令之前，您需要单独运行该命令以获得更高的权限)。

sudo groupadd mynewgroup

将现有用户帐户添加到组

要将现有用户帐户添加到系统上的组，请使用usermod命令，将examplegroup替换为要将用户添加到的组的名称，将ampleusername替换为要添加的用户的名称。

usermod -a -G examplegroup exampleusername

例如，要将用户geek添加到组sudo，请使用以下命令：

usermod -a -G sudo geek

更改用户的主组

虽然一个用户帐户可以是多个组的一部分，但其中一个组始终是“主要组”，其他组是“次要组”。在用户的登录过程中，用户创建的文件和文件夹将分配给主要组。

要更改分配给用户的主组，请运行usermod命令，将examplegroup替换为您要作为主组的名称，并将exampleusername替换为用户帐户的名称。

usermod -g groupname username

请注意这里的-g。当您使用小写的g时，您将分配一个主组。如上所述，当您使用大写-G时，您将分配一个新的辅助组。

查看用户帐户分配到的组

要查看当前用户帐户分配到的组，请运行Groups：命令。您将看到组的列表。

groups

要查看与每个组关联的数字ID，请改为运行id命令：

id

要查看另一个用户帐户分配到的组，请运行Groups：命令并指定用户帐户的名称。

groups exampleusername

您还可以通过运行id命令并指定用户名来查看与每个组相关联的数字ID。

id exampleusername

组列表中的第一个组或id列表中“gid=”后显示的组是用户帐户的主组。其他组是次要组。因此，在下面的屏幕截图中，用户帐户的主要组是示例。

在一个命令中创建新用户和分配组

您有时可能希望创建一个可以访问特定资源或目录的新用户帐户，如新FTP用户。您可以在使用useradd命令创建用户帐户时指定将向其分配用户帐户的组，如下所示：

useradd -G examplegroup exampleusername

例如，要创建一个名为jsmith的新用户帐户并将该帐户分配给ftp组，您需要运行：

useradd -G ftp jsmith

当然，之后您需要为该用户分配密码：

passwd jsmith

将用户添加到多个组

将辅助组分配给用户帐户时，您可以通过用逗号分隔列表来轻松地一次分配多个组。

usermod -a -G group1,group2,group3 exampleusername

例如，要将名为geek的用户添加到ftp、sudo和example组，您可以运行：

usermod -a -G ftp,sudo,example geek

您可以指定任意数量的组-只需用逗号将它们全部分隔即可。

查看系统上的所有组

如果要查看系统上所有组的列表，可以使用getent命令：

getent group

此输出还将显示哪些用户帐户是哪些组的成员。因此，在下面的屏幕截图中，我们可以看到用户帐户syslog和chris是adm组的成员。
