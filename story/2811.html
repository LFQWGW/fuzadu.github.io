使用SQL Server Management Studio恢复SQL数据库备份

我们之前介绍了使用命令行执行的简单SQL数据库恢复，该命令行非常适合于恢复在同一SQL Server安装上创建的备份文件，但是，如果您要恢复在不同安装上创建的备份，或者只是喜欢使用点击式界面，则使用SQL Server Management Studio(或速成版)可以轻松完成此任务。

注意：SQL Server专家可能希望跳过今天的课程，因为它是针对初学者的。

在开始之前，您需要将SQL备份文件(通常扩展名为.BAK)复制到目标SQL Server计算机上的本地硬盘驱动器。

打开SQL Server Management Studio，然后登录到要将数据库还原到的SQL Server。最好以Windows管理员或SQL‘sa’用户身份登录。

登录后，右键单击“数据库”文件夹，然后选择“恢复数据库”。

单击“恢复源”部分下“从设备”旁边的省略号按钮。

将“文件”设置为备份媒体，然后单击“添加”。

浏览到要还原的SQL备份(BAK)文件。

在还原数据库对话框中，键入或选择要将此备份还原到的数据库的名称。

如果选择现有数据库，它将被替换为备份中的数据。 如果键入SQL Server安装中当前不存在的数据库名称，则会创建该名称。

接下来，选择要使用的还原点。由于一个SQL备份文件可以包含多个备份，因此您可能会看到列出了多个还原点。

此时，已经为要恢复的数据库输入了足够的信息。但是，SQL备份文件存储有关将数据文件复制到何处的信息，因此如果出现任何文件系统问题(例如目标目录不存在或数据文件名冲突)，将会出现错误。在还原在不同SQL Server安装上创建的备份时，这些问题很常见。

要查看和更改文件系统设置，请在恢复数据库对话框中单击左侧的选项页。

在选项页面上，您需要确保“还原为”列指向有效的文件夹位置(您可以根据需要更改它们)。文件不一定要存在，但是文件夹路径必须存在。如果各自的文件确实存在，SQL Server将遵循一组简单的规则：

如果“To database”(来自“General”页)与还原数据库备份(即还原到匹配的数据库)匹配，则在还原过程中将覆盖相应的文件。 如果“目标数据库”与还原数据库备份不匹配(即还原到另一个数据库)，则需要选中“覆盖现有数据库”才能完成还原过程。请谨慎使用此功能，因为您可能会将数据库备份信息还原到来自完全不同数据库的数据文件之上。

通常，您可以根据“原始文件名”(SQL Server用来引用各自文件的内部名称)来区分不同的数据库。

设置恢复选项后，单击确定。

结束 / 结尾 / 结论 / 推论

SQL Server Management Studio使还原过程变得简单，如果您很少执行数据库还原，则非常理想。此过程适用于从Express到Enterprise的所有SQL Server版本。如果您使用的是速成版，则可以下载SQL Server Management Studio Express以访问此界面。
