了解Windows Server 2008中的Hyper-V新功能

本文由Metrotek Solutions的MetroTek Geek撰写，Metrotek Solutions是DC领域的计算机帮助提供商。

Microsoft Windows Server 2008通过Hyper-V实现虚拟化

作为Windows Server 2008发布的一部分，微软通过Hyper-V正式开启了真正的Windows服务器虚拟化时代。过去，虚拟化只在亚马逊等公司的数据中心和他们的EC2计算云中才能找到。现在，微软为我们带来了虚拟化。Windows Server 2008和Hyper-V恰逢其时。随着处理器体系结构从提高速度转向多核，虚拟化将在不久的将来成为服务器空间的重要趋势。

多台服务器。一个盒子。内存很大。

Hyper-V是1类(本机)虚拟机管理程序，这意味着它在根级别运行，可以直接控制服务器的硬件。虚拟操作系统作为刚好位于Hyper-V堆栈上方的分区独立运行。使用服务器虚拟化，您实际上是在一台物理机上运行多台服务器。以前，Microsoft提供的唯一服务器虚拟化产品是Virtual Server 2005，它作为类型2(托管的)虚拟机管理程序在Windows Server 2003等宏操作系统上运行。这意味着Virtual Server 2005虚拟机管理程序不能直接访问硬件。Virtual Server2005中的来宾操作系统实质上是通过宏内核运行的线程化应用程序。Hyper-V取代了操作系统层，并作为真正的类型1微内核虚拟机管理程序运行。

运行Hyper-V有两个关键要求：x64版本的Windows Server 2008和具有硬件辅助虚拟化扩展的64位处理器(如Intel VT或AMD-V系列处理器中的那些)。虽然Hyper-V需要64位工作环境，但来宾操作系统可以是32位或64位。支持的来宾操作系统(您想要虚拟化的操作系统)包括*：

Windows Server 2008。 Windows Server 2003。 Windows 2000 Server。 WindowsVista。 Windows XP Professional。 SUSE Linux Enterprise Server 10*其他操作系统也可以运行，但在官方身份下不受支持

您可能已经注意到，Hyper-V需要64位环境。考虑到在32位体系结构中发现的4 GB内存限制，这并不令人惊讶。在多个实例上划分4 GB空间不再起作用。幸运的是，带有Hyper-V的Windows Server 2008企业版和数据中心版可以支持每个虚拟实例高达2 TB的物理内存和64 GB的内存。那可是很大的内存啊！

快速迁移和高可用性

虚拟化服务器几乎可以立即从快速从一台物理服务器迁移到另一台物理服务器中受益。带快速迁移的Microsoft Hyper-V可以将来宾虚拟操作系统的状态保存到共享存储，将存储连接从一台服务器移动到另一台服务器，然后将映像恢复到新服务器。快速迁移与Windows Server 2008企业和数据中心群集服务协同工作，为您的群集服务器提供高可用性和快速灾难恢复功能。无论是计划内(维护)还是计划外(灾难恢复)，虚拟化技术的使用都会增加服务器的正常运行时间和可用性。

虚拟设备

虚拟设备是虚拟化提供的一个很有前途的新开发。供应商可以将操作系统和运行其产品所需的所有软件打包成虚拟的“黑盒”映像。然后，即使配置不同，也可以在不同的计算机上部署此映像。虚拟设备可以降低与管理和部署多层软件相关的安装、配置和维护成本。微软虚拟硬盘(VHD)测试驱动程序就是一个很好的例子。从VHD网站上看，“Microsoft虚拟硬盘(VHD)格式是通用的虚拟化文件格式，它提供统一的产品支持系统，并为客户提供更无缝的可管理性、安全性、可靠性和成本效益。VHD格式在单个文件中捕获整个虚拟机操作系统和应用程序堆栈。“。随着虚拟化变得更加普遍，我们将开始看到更多创新和有用的虚拟设备。

到V或不到V

服务器虚拟化代表着高效服务器空间发展的下一个合乎逻辑的步骤。随着服务器开始转向多核架构和64位平台，虚拟化应该成为优化服务器工作负载性能和降低成本的一个越来越重要的因素，特别是在服务器扩散成为一个日益严重的问题的情况下。曾经严格限制在数据中心和开发实验室范围内的东西现在正在成为IT现实。虽然有一些性能问题需要考虑，但您可能会发现虚拟化的优势非常值得投资。
