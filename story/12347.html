Windows10的感恩节周末充满了更新漏洞

生活中没有什么是确定的，除了死亡、税收和Windows10更新错误。当我们享受假期时，微软承认它破坏了从Windows Media Player到一些PC上的声音输出的一切。

更多损坏的英特尔驱动程序更新

在美国东部时间11月21日下午5点整，也就是感恩节的前一天，微软更新了Windows 10更新历史记录页面，指出了2018年10月更新的一个新问题。在某些采用英特尔显卡的系统上，通过HDMI、USB-C或DisplayPort连接的显示器或电视的音频播放中断。

微软表示，这是因为英特尔向个人电脑制造商发布了错误版本的显示驱动程序。那些PC制造商意外地打开了Windows中不受支持的功能。微软随后发布了这些驱动程序更新，但在此之前没有发现这个问题。换句话说，微软、英特尔和个人电脑制造商都搞砸了。

Windows 10不会在搭载该硬件的系统上安装2018年10月的更新，因为现在已经有了“更新块”。“微软正与英特尔合作，让这些驱动程序”过期“。如果更新已安装在您的系统上，并且损坏了您的声音，下面介绍如何修复它。

请注意，这与10月份损坏某些PC上的英特尔声音的坏驱动程序更新不同。感谢伍迪·莱昂哈德(Woody Leonhard)注意到这一点。

Windows Media Player无法查找

微软在2018年10月的更新中也打破了Windows Media Player。在感恩节周末，微软更新了一份支持文档，指出11月13日发布的一项更新打破了Windows Media Player中的搜索栏。该文件称“微软正在制定解决方案，并将在即将发布的版本中提供更新。”换句话说，谁知道这个问题什么时候能解决。我们希望您不要使用Windows Media Player！

同一份文件还说，我们报告的默认应用程序错误仍然没有修复。微软仍表示，修复“将在2018年11月下旬推出”，但时间已经不多了。

较新的Microsoft Update错误

这些只是感恩节周末的虫子，但最近有一些较小的问题没有引起注意。

11月16日，微软MVP Sumit Dhiman在Microsoft Answers网站上表示，他看到几台PC无法安装2018年10月的更新，因为它们启用了开发者模式。他推荐的修复方法是，Windows用户应该在更新之前禁用开发人员模式。如果不这样做，升级过程可能总会在中途失败。

不仅仅是Windows10，微软最近发布了名为KB 4461529的Outlook2010更新。11月16日，微软承认这一更新导致许多64位Outlook 2010应用程序在启动时崩溃，正如首席执行官Woody Leonhard所发现的那样。谢天谢地，这个问题已经解决了。

Windows10的开发过程显然被打破了

这变得越来越荒谬了。映射的网络驱动器和苹果的iCloud在2018年10月的更新中也仍然出现故障。微软承诺将更多地谈论质量问题，但没有宣布对Windows 10的开发流程进行具体修改。微软并没有追求重大的质量变化，而只是不断尝试在Windows上添加更多广告，并致力于开发一些可爱的小功能，比如一个轻巧的主题。

让我们再说一遍：Windows不是一项服务；它是一个操作系统。微软早就应该放慢脚步了。
