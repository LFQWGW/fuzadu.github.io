复杂度

停止狩猎，开始寻找吧！

把你所有的东西都放在你的电脑上是很棒的，直到你找不到为止。你的硬盘上可能潜藏着大量的照片、文档、音乐，甚至视频，以至于你可能甚至不知道你拥有的是什么。如果你只需在搜索框中输入几个词，你的东西就会神奇地出现，那不是很棒吗？

实际上，你可以，而且你过去可能经常使用Windows搜索功能。使用搜索，您可以完成所有操作：查找文件、定位设置，甚至启动应用程序。但我们真正想要掌握的是发现和定位。毕竟，知道你的电脑上某个地方有一些东西，但不知道在哪里可以找到，这真的很令人沮丧。

本How-to Geek School系列自上而下地介绍Windows Search-从开始搜索、桌面搜索，以及如何使用高级查询语法创建更好的搜索，以便您可以快速高效地查找材料。

我们将从一些基本的Windows搜索概念开始，然后以讨论搜索索引器(包括如何对其进行微调和扩展)来结束今天的课程。让我们先简要讨论一下Windows搜索是如何在过去几年中发展起来的。

Windows Search的发展演变

从Windows95开始，Windows搜索已经取得了相当大的进步，说得委婉些。老实说，搜索那些早期的Windows版本有点像个笑话。这是基本的、有限的，而且绝对无济于事。公平地说，在任何Windows9x版本(包括令人难以置信的糟糕的Windows ME)上，搜索都是事后才想到的。

作为Windows95或Windows98用户，在您的计算机上查找内容非常简单和缓慢。当你花了一段时间通过28.8调制解调器下载东西，却忘了把它放在哪里的时候，你基本上可以把它保留下来。随着计算机硬盘变得越来越大，存储在硬盘上的文件也越来越多，使用旧的搜索方法并不能减少麻烦。

Windows Search在其当前版本中描述了Vista和所有后续版本中的搜索设备。在此之前，在Windows XP和Windows Server 2003中，它被称为Windows桌面搜索(WDS)，也可以作为可下载的独立更新提供给Windows 2000用户。WDS预示着索引搜索的到来。

索引搜索意味着计算机在您的文件和设置中爬行，并建立索引-基本上是一个告诉搜索所有内容在哪里的大列表。您可能会认为它是真实书籍中的实际索引，只有此索引才能提供在您键入时显示的即时结果，从而节省时间和击键。

早期Windows XP时代的索引服务最大的问题之一是它太多太快了。当时的计算机性能仍然相对较差，在使用消耗系统资源的索引文件时表现不佳，导致许多计算机干脆完全禁用索引。请记住，在2000年代初到中期，系统RAM和CPU仍以兆赫为单位测量，系统硬盘驱动器由连接到并行接口的缓慢旋转的柱面组成，与今天的行业标准串行接口相比，该接口完全是冰川般的。

今天，这不再是一个问题，因为计算机现在可以处理你扔给它们的任何索引。它发生的时机再合适不过了，因为如果你的电脑和我们的电脑有任何相似之处，你就有足够的时间把你可能早就忘记的东西塞满它，但有时你可能需要找到它。

很高兴知道，如果使用正确的工具和知识，您可以非常快速、轻松地找到所有内容。这就是Windows搜索的用武之地，它可以找到你硬盘上的任何东西，如果你知道如何查找的话。

基本Windows搜索概念

让我们介绍一些搜索基础知识和概念，以便更好地掌握本系列将要介绍的内容。

开始搜索

目前，您可以通过两种方式从头开始搜索，但它们或多或少都做了同样的事情。还有一种老式的方式，即开始菜单底部的搜索栏，比如在Windows7中。

还有一种新奇的Windows8方式，当你点击Win+S(或其他键盘组合)或点击搜索咒语时，搜索功能会从左屏幕边缘滑出。

正如我们所说，虽然新的界面有一些新的花哨，但这两个界面-Windows8和之前的版本-基本上做了同样的事情。我们将在明天的第2课中更详细地介绍开始搜索，不过我们将主要集中在新的Windows8方式上。

桌面搜索

当我们提到桌面搜索时，我们主要谈论的是从任何文件资源管理器窗口进行搜索的能力。

在Windows Vista之前，在Windows中的搜索仅限于从开始菜单进行搜索。在当前版本的Windows中，您可以打开任何文件资源管理器窗口并在该位置进行搜索。

事实上，当您以这种方式搜索时，会出现一个特殊的上下文工具栏或功能区，为用户提供更多选项。在第3课中，我们将从上到下介绍桌面搜索。

布尔运算符

任何使用互联网查找资料的人都可能熟悉布尔运算符。布尔运算符的示例包括AND、OR和NOT。

Boolean允许用户在搜索词之间创建关系，允许您将它们串在一起以创建有针对性的查询。这可能会扩大您的搜索范围，也可能会聚焦您的搜索。您可以这样使用Boolean，这样您就可以编写表达式，从而可以搜索图像文件，但不能搜索特定类型的图像文件，也可以搜索Microsoft Word文档和纯文本文件。

有很多关于布尔运算符的知识需要学习，因此我们将在第4课中详细介绍它，但在本系列中我们会不时地提到它。

诠释数据

元数据是关于数据的数据。这不仅仅是美国国家安全局收集的东西，它实际上可以快速完成任何搜索，其中你知道文件是关于什么的，但你只是不确定其他属性。

当您拍摄照片、保存文档或发送电子邮件时，所有其他类型的数据都会随其一起记录，或者可以在以后添加到其中。考虑一下任何一张典型的照片。虽然提取数字照片的基本属性(如分辨率、创建日期和其他基本属性)似乎是合乎逻辑的，但当您得知描述该文件的方式要多得多时，您可能会感到震惊。

如你所见，文件中默认记录了很多信息，比如曝光、ISO设置、相机的型号和型号等等。如果你使用照相手机，它很可能也会记录你的位置。关键是，在进行搜索时，您可以使用很多矢量来磨练。

此外，如果您进一步向下滚动图像属性，您可以看到还可以添加有关照片的数据。

当您尝试使用特定信息查找和排序文件时，所有这些额外的元数据都非常有用。问题是，您通常必须自己添加扩展信息，这可能既耗时又费力。

出于本系列的目的，我们将重点介绍默认情况下通常在文件中找到的元数据。

高级查询语法

简而言之，高级查询语法(AQS)是一组复杂的规则，您可以在Windows搜索中键入这些规则。这些规则遵循四个文件参数：属性、种类、存储和内容。我们将在第4课和第5课中更详细地解释所有这些内容，并给出一些有价值的例子，但目前，我们只想对每一课做一个粗略的概述。

文件属性

您可能至少相当熟悉文件属性，这些属性以多种方式描述文件。例如，文件的大小是一个属性，它的创建、修改和上次访问的日期也是一个属性。

文件类型

说到文件类型，文件类型应该很容易理解。每个文件都是一种类型，您有图像文件、视频文件、文档文件等等。

文件存储

位置称为“商店”，也可以包括数据库。例如，如果要搜索电子邮件，可以搜索Outlook数据库或Documents文件夹中的特定Microsoft Office文件。老实说，您通常可以在文件资源管理器中浏览到某个位置，但这使您能够从任何地方搜索任何位置，而不必转到该特定位置。

文件内容

最后，你可以通过文件的内容，简单地说，文档中的关键字来搜索，所以如果你还记得20年前你在大学里写的一篇论文，其中你说“莎士比亚就是炸弹”，你可以通过搜索文档(文件种类)及其内容来快速而容易地找到它。

搜索索引器

Search Indexer是一项在后台运行的服务，它“为文件、电子邮件和其他内容提供内容索引、属性缓存和搜索结果”。您可以打开任务管理器，并注意搜索索引器对系统的影响。

可以肯定地说，除非搜索索引器真的在您的硬盘上主动爬行并索引内容，否则它对您的系统的影响将微乎其微，因此您可能不应该禁用它。也就是说，如果您热衷于充分利用系统的所有性能，您可以打开服务管理控制台并永久禁用搜索索引器。

对于本系列而言，显然搜索索引器是绝对必要的。在下一节中，我们将讨论如何微调和扩展索引器。

配置和扩展搜索索引器

在结束今天的课程之前，我们想花一点时间来介绍一下搜索索引器，即配置和扩展(添加位置和文件类型)它。

您可以使用控制面板中的索引选项配置搜索索引器。

每当Windows启动时，索引器都会启动。第一次运行时，它将爬行您的硬盘，并索引其配置为要查找的所有位置。当您添加或删除文件时，索引器将自动更新，这样您就看不到不存在的结果，而可以找到存在的内容。

当计算机空闲时，索引器将全速运行，而当您使用它时，索引器速度很慢，因此，如果您希望它为所有内容编制索引，以便开始获得完整的结果，那么您需要让计算机单独运行，这样它才能运行到完成。

在上面的屏幕截图中，我们看到的是默认情况下的完全索引系统。显然，如果您的系统上没有Outlook，您将看不到这一点，但是您应该明白这一点。请注意，“Users”文件夹将包含所有特殊文件夹，如Documents、Video、Music等。

但是，假设您有其他想要索引的位置。在这种情况下，您可以扩展索引以查看存档和其他数据存储。例如，假设您添加了一个外部驱动器，其中包含您不想在较小的系统驱动器上保留的所有内容。显然，您想要一种快速找到该驱动器上的内容的方法，对其进行索引是一个完美的解决方案，但索引器不会在新位置查找，除非您特别要求它这样做。

要添加位置，您需要单击“修改”，将打开一个新屏幕，显示您所有的可索引位置。在那里，您可以选择要添加的任何新位置。

一旦您添加了您的一个或多个位置，您可以单击“确定”，索引服务将合并新的信息。

高级选项

在索引选项屏幕上，您还会看到一个“高级”按钮。在“高级选项”窗口上，有两个选项卡：“索引设置”和“文件类型”。

索引设置选项卡允许您关注几件事。您可以搜索索引加密的文件，这可能会构成安全风险(因为加密的文件通常是有原因的)。

您还可以让索引器区分带变音符号的单词和不带变音符号的单词。这意味着，例如，“naive”和“naive”或“manana”和“mañana”是分开索引的，而不是集中在同一搜索结果中。

“重建”按钮允许您丢弃当前的索引器，并在遇到任何问题时创建新的索引器。请记住，如果您这样做，您将不得不等待索引器完成其工作，然后才能再次彻底搜索。

最后，如果您想要将索引移动到新位置(例如，移动到性能更好的驱动器或空间更大的驱动器)，则可以使用“索引位置”设置来执行此操作。只需单击“选择新的”并浏览到您希望索引驻留的新位置。在新位置生效之前，您需要重新启动服务(或系统)。

添加和删除文件类型

您可以通过添加新的文件类型来进一步扩展索引的覆盖范围。为此，您需要单击“文件类型”选项卡，在框中键入新文件类型的扩展名，然后单击“添加”按钮。

此外，通过取消选中要排除的每个文件类型旁边的框，可以从索引器中删除文件类型。

您还可以更改文件的索引方式。如果要为文件的属性和内容编制索引，只需选择要影响的每种文件类型，然后单击该选项即可。默认情况下，大多数文件都已以某种方式设置。例如，Word文档将对其文件内容进行索引，而图像文件则不会。

在很大程度上，您会发现系统上的每种文件类型几乎总是已经以这样或那样的方式编制了索引。

结束 / 结尾 / 结论 / 推论

今天的课程应该会让你坚定地指向明天关于开始搜索的课程，同时让你熟悉本周将学到的概念。你今天的作业是配置你的索引设置，选项，如果需要的话，用你可能想要包括的位置扩展你的索引器。请记住，如果您不喜欢索引器的运行方式，您可以随时单击“重建”并重新开始，所以请继续胡乱操作，而不必担心会损害您的系统！
