如何将Windows Server试用期无缝延长至240天

他们产品的Microsoft评估版本是非常有价值和有用的工具，因为它们允许您免费使用数量不限的测试、演示和开发环境。唯一的问题是评估版本是有时间限制的，所以您从评估版本中挤出的时间越多，它们就越有用。

在这里，我们将向您展示如何最大限度地延长Windows Server 2008 R2评估版本的使用时间。

它是如何工作的

一旦激活，Windows Server 2008 R2评估(或试用)将持续180天，之后您必须输入有效的许可证密钥，否则服务器将在使用一小时后自行关闭。

对于大多数需要激活的Microsoft产品，在需要激活之前都有几天的宽限期，在此期间产品可以使用(不受限制)。对于Windows Server 2008 R2评估，此宽限期为10天，在需要激活之前可以重置(或重新武装)5次。充分利用时，可额外提供60天的使用时间。

为了让您放心，这种做法是完全合法的，而且实际上是由微软推动的。我们只是通过一个自动的“设置并忘记”它的过程来实现这一点。

重新武装恩典时期

我们的第一个任务“重置激活计时器”利用了10天宽限期中的全部5天。此任务的每个实例都是一次特定日期的呼叫。第一次调用应设置为从第一次系统引导起10天，每次后续调用设置为前一次调用后10天。

注意：我们已经为您提供了计算此信息的工具，本文稍后将对此进行讨论。

实际的命令是使用-rearm参数调用“slmgr.vbs”脚本，然后重新启动系统。

激活服务器

一旦我们用完了所有的重置/重新武装命令，我们就必须实际激活服务器。这在“激活服务器”任务中完成。

此任务应设置为从上次调用到“重置激活计时器”任务的10天。

实际的命令再次调用“slmgr.vbs”脚本，但带有-ato参数(执行激活)，然后重新启动。

最终结果是180天时间限制的服务器激活。

导入计划任务

虽然您可以手动创建适当的计划任务，但为方便起见，我们以可导入的XML格式包含了这些任务(请参阅底部的链接部分)。

要导入这些文件，只需打开Task Scheduler，右键单击Task Scheduler库项目，然后选择Import Task。

浏览到Extracted Tasks文件夹并选择一个。

选择导入后，您需要更改任务运行的帐户以使其与您的服务器保持一致。在“创建任务”对话框(拾取要导入的任务后打开)中，单击“更改用户或组”按钮。

输入“管理员”，然后单击Check Names(检查名称)，然后单击OK(确定)。

您应该会看到与您的计算机对应的管理员帐户为有效帐户。

当您应用任务的设置时，系统将提示您输入管理员密码。

对每个XML任务文件重复这些步骤。

过期警告电子邮件

240天后，服务器激活将到期。虽然您可以通过登录服务器查看剩余时间，但8个月后很容易忘记执行此操作。因此，为了自动提醒您即将到期的过程，我们包含了一个计划任务，该任务将在到期临近时通过每日电子邮件自动警告您。

但是，为了使其正常工作，您必须在系统上配置BLAT(命令行电子邮件工具)。

在Add Arguments行下，滚动到末尾，并将-to参数后面的值更新为警告应发送到的电子邮件地址。此外，如果您要编辑生成的电子邮件的邮件正文和/或主题，也可以在此处执行此操作。
