如何在Windows Server 2012中关闭和打开GUI

当Server Core最初发货时，许多Windows管理员都避免使用它，因为你只能使用命令行，但随着Windows Server 2012的出现，这一点发生了变化，Windows Server 2012支持使用混合模式。

关闭GUI

在Windows Server8中，GUI保持了最新Windows Server操作系统的模块化特性，进而成为一项“功能”。这使得删除GUI变得非常容易。要开始使用，请启动服务器管理器。

单击Manage(管理)，然后从菜单中选择Remove Roles或Feature(删除角色或功能)。

单击“下一步”跳过“开始前”页，然后从服务器池中选择您的服务器并单击“下一步”。

由于GUI不是角色，我们可以再次单击Next跳过角色部分。

当您到达功能页面时，您需要取消选中“User Interfaces and Infrastructure”选项旁边的复选框，然后单击Next。

现在勾选“重新启动目标服务器”框，然后单击“删除”。

现在将删除GUI。

删除二进制文件后，您的服务器将自动重新启动。

一旦它重新启动，并且您登录，您将只能使用命令行。

打开GUI

一旦GUI关闭，您就会想知道如何将其取回。为此，我们使用SConfig，因此请继续在命令行中键入SConfig，然后按Enter键。

您可以在屏幕底部附近看到，我们可以使用“12”恢复GUI，因此键入12并按Enter键。

系统将警告您启用GUI需要重新启动，然后单击YES按钮。

这将启动DISM，它将开始为GUI Shell添加二进制文件。

完成后，系统将询问您是否要立即重新启动计算机，键入“y”，然后按Enter键重新启动。
