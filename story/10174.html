如何在Windows、Mac和Linux上查找Chrome配置文件文件夹

您的Chrome配置文件存储您的浏览器设置、书签、扩展、应用程序和保存的密码。您的个人资料存储在计算机上的一个单独文件夹中，因此如果Chrome出现问题，您的信息将被保存。

如果你曾经遇到Chrome的任何问题，尝试一个新的用户配置文件可以帮助你排除故障。一些调整可能还需要你手动定位和编辑你的Chrome个人资料，所以知道它在哪里是很方便的。

Chrome的默认配置文件文件夹的位置因您的平台而异。地点为：

Windows 7、8.1and 10：C：\Users\<username>\AppData\Local\Google\Chrome\User Data\Default。 Mac OS X El Capitan：Users/<用户名>/Library/Application Support/Google/Chrome/Default。 linux：/home/<username>/.config/google-chrome/default

只需将<username>替换为您的用户文件夹的名称。默认配置文件文件夹仅命名为Default(或Linux中的Default)。但是，如果您创建了其他配置文件，它们的文件夹名称就不那么明显了。创建配置文件时分配给配置文件的名称显示在Chrome窗口标题栏右侧的名称按钮上。不幸的是，Chrome在关联的配置文件文件夹中使用的名称是一个通用的编号名称，如“Profile 3”。

如果您需要编辑您的其他配置文件之一，您可以非常简单地计算出它的文件夹名称。每次切换配置文件时，都会使用该配置文件打开一个新的Chrome窗口。在显示您要查找的名称按钮上的配置文件的Chrome窗口中，在地址栏中输入Chrome：//版本号，然后按Enter键。

“配置文件路径”显示当前配置文件的位置。例如，我的“工作”配置文件在Windows10中的位置实际上是C：\Users\Lori\AppData\Local\Google\Chrome\User Data\Profile3。您可以选择路径并将其复制并粘贴到Windows中的文件资源管理器、OS X中的Finder中，或者粘贴到Linux中的Nautilus之类的文件管理器中以访问该文件夹。

要备份配置文件，请将Windows上的userdata文件夹、Mac OS X El Capitan上的Chrome文件夹或Linux上的Google-Chrome文件夹中的默认配置文件文件夹和任何编号配置文件文件夹复制到外部硬盘或云服务。您可以通过删除(或重命名或移动)数据(用户数据、Chrome或Google-Chrome)文件夹来完全重置Google Chrome。下次启动Chrome时，将使用新的默认配置文件创建一个新的数据文件夹。

如果你真的想动手，你可以用不同的浏览器设置、书签、扩展、应用程序和保存的密码来设置多个配置文件。如果你想测试扩展之类的东西，或者解决Chrome中的问题，而不想弄乱你的主配置文件，这是很有用的。你甚至可以对不同的用户有不同的个人资料，或者像“工作”和“个人”这样的不同情况。
