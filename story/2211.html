在VMware Server上安装Windows Home Server“Vail”

如果您是Windows Home Server的狂热爱好者，您很可能知道名为“Vail”的新Beta代码。如果您想测试它而不用担心有额外的机器，我们可以在VMware Server中免费安装和使用它。

之前，我们向您展示了如何在VMware Workstation上安装WHS Vail。今天，我们将看看如何使用VMware Server2和免费的Vail Beta ISO完全免费安装它。

开始

您需要以下项目才能开始使用。

VMware Server 2-需要注册。 WHS Vail Beta ISO-通过Microsoft Connect。 能够实现硬件虚拟化的计算机，运行64位版本的Windows。 主机上有足够的硬盘空间来创建160 GB的虚拟硬盘，这是系统的最低要求。 主机上至少有2 GB或更多RAM-分配给Vail的最小RAM为1 GB

在本文中，我们使用了一台配备酷睿i3处理器和6 GB RAM的计算机，运行Windows 7旗舰版x64和VMware Server 2

验证硬件虚拟化

在我们的上一篇文章中，我们向您展示了易于使用的免费实用程序安全，以确定您的计算机是否能够进行硬件虚拟化。

微软还有另一个免费的实用工具，可以做同样的事情，叫做微软硬件辅助虚拟化检测工具(链接如下)。与安全文件一样，它不需要安装，只需以管理员身份运行可执行文件即可。

系统将提示您同意EULA，之后您将收到如下所示的消息。显示您的计算机是否能够进行硬件虚拟化。

如果它不支持硬件虚拟化，很多时候该功能会被锁定，你需要仔细检查你的BIOS才能找到打开它的设置。如果这不起作用，请确保您的BIOS有最新的更新。

下载并安装VMware Server

如果您尚未安装，则需要下载并安装VMware服务器。它是免费的，但需要注册(链接如下)。

注意：在开始安装之前，请注意，如果您已经安装了VMware Workstation，系统将提示您先卸载它。

下载文件后，按正常方式开始安装，在应该可以使用默认值的位置开始安装。

您注册后，他们发送电子邮件激活您的许可证并下载服务器，从该页面输入他们提供的序列号。

现在，您将能够从开始菜单以及它提供的其他工具访问VMware Server。要启动它，请单击VMware Server主页，该主页将在您的默认浏览器中本地启动。

下一步非常重要。您会注意到，在安装过程中，从未要求您输入用户凭据。默认情况下，它使用运行它的计算机的登录信息。而不是您的VMware登录信息。一开始这有点混乱，所以当您想要登录时请记住这一点。

登录后，您将看到摘要页面。现在我们已经准备好安装Windows Home Server“Vail”了！

创建虚拟机

好了，现在我们已经安装了VMware服务器并登录了，我们想要创建一个新的虚拟机来运行Vail。单击虚拟机\创建虚拟机。

创建虚拟机向导启动，您首先要为其命名。

接下来，我们需要选择来宾操作系统。从版本下拉字段中选择Microsoft Windows Server 2008(64位)。

现在选择您希望服务器使用的内存量。请记住，这将使用您主机的RAM，因此根据您的内存大小，您可能希望使用建议的1 GB大小。因为我们的机器有额外的内存，所以我们把它提升到了2 GB。您也可以在创建机器后随时更改这些设置。

在下一步中，我们需要创建一个新的虚拟磁盘。

现在我们需要分配它的容量。Vail的最小大小是160 GB，所以我们就采用这个大小。在我们的测试中，我们通过选中现在分配所有磁盘空间获得了最好的结果。

接下来，在网络适配器下，单击添加网络适配器…

在“属性”下，如果使用桥接网络连接，则效果最佳。这样，虚拟服务器将被分配其自己的IP地址，并将像您网络上的独立物理机一样运行。

现在，您需要将Vail Install DVD、Restore CD和SrvRecovery ISO文件复制到C：\Virtual Machines，以便VMware服务器可以在我们加载它时看到它。VMware Server中的浏览功能不允许浏览您的本地目录，如Workstation或Player。

对于CD/DVD驱动器，选择使用ISO映像(这是Vail下载为的格式)。你不需要先把它刻录到光盘上，…。除非你出于某种原因想这么做。当然，如果您是在一台实际的机器上安装它，您会希望将其刻录到光盘上。

单击浏览按钮…

现在我们可以选择VailInstallDVD.iso并单击OK。

然后返回到Properties屏幕，确保选中Yes to Connect at Power On，然后单击Next。

单击不要添加软盘驱动器，除非您出于某种原因想要添加…

对于USB控制器，您可以添加对它的支持或不添加，这取决于您。出于测试目的，我们将继续并单击Add a USB Controller。

现在我们已经完成了VM的设置。查看概述并验证其配置是否正确。如果没有，您可以在这一点上返回并进行调整。如果需要，您还可以在此时添加更多硬件，但是现在一切都配置好了，可以安装Vail，所以我们将保留现有的硬件。请记住，我们也可以在安装Vail之后更改设置。

选中现在打开新虚拟机的电源，然后单击Finish按钮。

现在请稍候，等待构建新的虚拟服务器。所需的时间长度将根据您的计算机的硬件类型而有所不同。在我们的系统上，创建虚拟硬盘大约需要30分钟。

成功创建驱动器并打开虚拟机后，在Inventory下单击Vail VM，然后单击Console选项卡。

您将看到一条消息，提示您需要安装VMware Remote Console Plug-in…。只需单击即可安装。

起初我们通过Firefox运行VMware服务器，安装插件后遇到了一些问题。

安装插件后，您需要重新启动Firefox。当我们回来的时候，我们无法启动控制台。这可能与插件或某些缓存问题有关。我们会想出一个解决办法，稍后再发表一篇关于它的文章。

我们希望安装Vail，因此我们在Internet Explorer8中运行VMware Server。同样，您需要安装远程控制台插件。

IE的安装是按照安装向导直接进行的。

当您第一次在IE中转到您的服务器时，您将收到有关安全证书的警告，请单击“继续到此网站”。

在您登录…之后。出现安全栏时选择Run Add-on以启动远程控制台插件。

安装WHS Vail

从VMware Server Web主页中选择Vail Server，然后单击Console选项卡以打开VMware Remote Console。

VMware Remote Console随即打开，您现在可以开始安装Vail了。在我们的系统上，安装大约需要45分钟才能完成。

在安装过程中，远程控制台将重新启动几次。

经过长时间的等待，并在安装过程中输入了一些它要求的信息，我们就有了新的WHS Vail虚拟服务器！

安装VMware工具

一旦Vail安装完成，我们需要添加VMware工具以获得更流畅的体验。从Web界面的Status字段中，单击Install VMware Tools

将弹出一条解释VMware Tools的消息，单击Install(安装)按钮。

在远程控制台中，您将看到自动播放弹出…。单击Run setup.exe(运行setup.exe)。

然后运行安装向导，接受默认值。

需要重新启动虚拟机服务器才能完成安装过程。

重新启动虚拟机后，您将看到Ctrl+Alt+Del屏幕。

单击VMware Remote Console\Troubleshoot\Send Ctrl+Alt+Del。

然后登录到您的WHS Vail虚拟服务器。

结束 / 结尾 / 结论 / 推论

如果您正在寻找一种方法来测试新的Windows Home Server“Vail”Beta，那么使用VMware Server是一个免费完成测试的好方法。您将能够RDP进入它，就像它是您网络上的一台单独的物理机一样。要测试将计算机连接到它并使用Dashboard或Launchpad等新功能，您可以创建一个Windows VM，或在您的网络上连接一台额外的计算机。请确保该计算机尚未安装WHS版本1连接器软件，因为您不能在同一台计算机上同时安装这两个软件。

如果您当前运行的是WHS版本1，您不会想要用Vail测试版取代它。仍然有几个怪癖需要首先解决。这与Windows7Beta不同，在Windows7Beta中，你可以在你的机器上安装它，几乎没有问题。至少您可以探索Vail，看看最终版本发布后会发生什么。
