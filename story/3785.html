IT：如何在Windows Server 2008 R2上安装Active Directory

Active Directory对于构建在客户端-服务器网络模型上的任何Microsoft网络都是必不可少的-它允许您拥有一个称为域控制器(DC)的中央服务器，该服务器为您的整个网络执行身份验证。他们不是登录到本地计算机，而是根据您的DC进行身份验证。让我们来看看如何安装微软的Active Directory。

就任 / 安顿 / 安装 / 就职

打开Server Manager并单击Roles，这将在右侧调出Roles Summary，您可以在其中单击Add Roles链接。

这将打开Add Roles Wizard，您可以在其中单击Next查看可用角色列表。从列表中选择Active Directory域服务，您将被告知需要添加一些功能，单击添加所需功能按钮，然后单击下一步继续。

将显示Active Directory的简要介绍以及指向其他资源的几个链接，您只需单击“下一步”跳过此处，然后单击“安装”开始安装Active Directory的二进制文件。

安装完成后，您将看到一条成功消息，只需单击“关闭”即可。

配置 / 构造 / 布局 / 外形

打开“服务器管理器”，展开“角色”，然后单击“Active Directory域服务”。在右侧单击Run the Active Directory Domain Services Installation Wizard(dcproc.exe)(运行Active Directory域服务安装向导(dcproc.exe))链接。

这将启动另一个向导，这次要配置域的设置，请单击“下一步”继续。

现在显示的消息与不支持Server 2008 R2支持的新加密算法的旧客户端有关，这些算法在Server 2008 R2中默认使用，请单击下一步继续。

选择在新林中创建新域。

现在您可以命名您的域了，我们将使用.local域，原因将在下一篇文章中解释。

由于这是我们域中的第一个DC，我们可以将林功能级别更改为Server2008R2。

我们希望在安装中包含DNS，因为这将允许我们拥有AD集成DNS区域，当您单击下一步时，系统将提示您一条消息，只需单击是继续。

您需要选择存储日志文件的位置，最佳做法是将数据库和SYSVOL文件夹存储在一个驱动器上，将日志文件存储在单独的驱动器上，但由于这是在实验室环境中，我将把它们全部保留在同一驱动器上。

选择强Active Directory还原模式密码，然后单击两次下一步开始配置。

通过查看下面的框，您将能够看到正在安装哪些组件。

完成后，系统将通知您并要求您重新启动PC。

伙计们，这就是我们要做的全部事情，现在您已经可以正常安装Active Directory了。
