复杂度

用户帐户、组、权限及其在共享中的角色

本How-to Geek School课程面向拥有至少一台Windows PC或设备且拥有自己的家庭网络的人员。最终目标是为您提供在Windows中设置共享所需的知识，并能够与家庭网络中的其他PC或设备共享文件、文件夹和设备，而无需考虑操作系统。

在第一课中，您将了解什么是用户帐户，以及用户帐户最重要的特征是什么。然后，我们将详细介绍Windows中可用的用户帐户类型，以及Windows 8与Windows 7中与此角度的不同之处。正如您将看到的那样，最新版本的Windows添加了与您过去习惯的用户类型非常不同的新用户类型。

接下来，我们将解释用户组和权限的概念。正如您将看到的，这些概念在与网络上的其他人共享时非常重要。

最后但并非最不重要的一点是，我们将解释在Windows8中使用Microsoft帐户与网络共享相关的好处。

我们开始吧!

什么是用户帐户？

用户帐户是设置和信息的集合，它告诉Windows您可以访问哪些文件和文件夹、您可以在计算机上执行什么操作、您的首选项是什么，以及在连接到网络时您可以访问哪些网络资源。

用户帐户允许您向Windows或任何其他操作系统进行身份验证，以便授予您使用它们的权限。多用户操作系统(如Windows)不允许用户在没有用户帐户的情况下使用它们。

在Windows中，您可以通过转到“控制面板”，然后转到“用户帐户和家庭安全设置>用户帐户”来管理计算机的用户帐户。

Windows中的用户帐户具有以下属性：

用户名-您为该帐户指定的名称。 密码-与用户帐户关联的密码(在Windows 7或更早版本中，您也可以使用空密码)。 用户组-共享相同安全权限的用户帐户的集合。用户帐户必须是至少一个用户组的成员。 类型-所有用户帐户都有一个类型，用于定义其权限以及在Windows中可以执行的操作。

Windows 7用户帐户

Windows 7及更早版本有三种重要的帐户类型：

管理者 / 行政官 / <律>遗产管理人

“Administrator”用户帐户拥有对PC的完全控制权。他或她可以安装任何内容并进行影响该PC所有用户的更改。

标准 / 榜样 / 水准

“标准”用户帐户只能使用管理员已安装的软件，并更改不影响其他用户的系统设置。

客 / 旅客 / 宿客 / <美>顾客

“Guest”帐户是一种特殊类型的用户帐户，其名称为Guest，没有密码。这仅适用于需要临时访问PC的用户。此用户只能使用管理员已安装的软件，不能对系统设置进行任何更改。

Windows 8用户帐户

除了Windows 7中已有的用户帐户外，Windows 8还引入了两种新类型的用户帐户：

Microsoft帐户

Microsoft帐户是具有关联电子邮件地址的用户帐户，使您可以访问所有Microsoft产品和服务。他们的密码总是不是空的。如果您使用的是outlook.com电子邮件地址(比方说Howtogeek@outlook.com)，则您有一个具有该地址的Microsoft帐户。

让事情更复杂的是，微软允许人们使用Gmail等第三方电子邮件服务创建微软账户。为简化操作，请记住，当您使用电子邮件地址登录Windows或任何Microsoft产品或服务时，您拥有Microsoft帐户。

Microsoft帐户可在多个系统和设备上运行。因此，您可以使用相同的帐户登录所有Windows 8.x设备、Xbox One控制台和Windows Phone。您不必为每台设备创建单独的帐户。

Microsoft帐户可以是管理员帐户，也可以是标准用户帐户。

本地帐户

本地帐户是本地存在的传统用户帐户，可以使用空密码。例如，在Windows 7中，所有用户帐户都是本地帐户。本地帐户可以是管理员帐户，也可以是标准用户帐户。它们只能在单个系统上运行，因此如果您有多个设备，则必须为每个设备创建单独的帐户。

用户帐户提供了一个额外的好处，即允许您与多个人共享同一台计算机，同时拥有您自己的文件和设置。每个人都可以访问他或她的用户帐户，而不会干扰其他人。

如何区分它们呢？

在Windows8.x中，您可以通过查看本地用户帐户是否使用电子邮件地址来快速区分本地用户帐户和Microsoft帐户。请看下面的屏幕截图，共享“管理帐户”窗口，可通过“控制面板>用户帐户和家庭安全设置>用户帐户>管理帐户”访问该窗口。

第一个帐户名为Ciprian Rusen，是一个Microsoft帐户。所有其他用户帐户都是本地帐户。Microsoft帐户是管理员，在其电子邮件地址下面用“Administrator”语句标记。所有其他用户帐户都是标准用户帐户，因为它们没有“Administrator”语句。

什么是用户组？

如前所述，用户组是共享相同安全权限的用户帐户的集合。

Windows有一个很长的预定义用户组列表，其中包括“管理员”和“用户”。但是，大多数预定义的用户组在管理员或第三方应用程序开始自定义它们之前没有用户帐户。用户组也可以由第三方软件和服务(如虚拟机)创建，这些软件和服务会创建隐藏的用户帐户和组，以便提供不同的功能或服务。

用户帐户是至少一个用户组的成员，而某些用户帐户是两个或更多组的成员，具体取决于它们的设置方式。

例如，设置为管理员的所有用户帐户都将是“管理员”组的一部分。标准用户帐户是“Users”组的一部分。但是，当您开始使用Windows中的家庭组网络功能时，这两种类型的用户帐户都将成为“HomeUsers”组的成员。

用户组由Windows自动管理，您不需要摆弄它们，即使您是管理员也可以。此概念非常重要，这样您就可以更好地了解文件共享的工作原理、权限分配方式等。

什么是文件和文件夹权限？

权限是将访问权限分配给特定用户帐户和用户组的一种方法。通过使用权限，Windows定义了哪些用户帐户和用户组可以访问哪些文件和文件夹，以及他们可以对这些文件和文件夹执行哪些操作。简单地说，权限是操作系统告诉您可以或不能对文件或文件夹执行哪些操作的方式。

要了解任何文件夹的权限，请右键单击该文件夹并选择“属性”。在“属性”窗口中，转到“安全”选项卡。在“组或用户名”部分，您将看到所有用户帐户和使用对该文件夹具有权限的组。如果选择一个组或用户帐户，请在“用户权限”部分查看其分配的权限。

在Windows中，用户帐户或用户组可以接收对任何文件或文件夹的以下权限之一：

读取-允许查看和列出文件或文件夹。查看文件夹时，您可以查看其所有文件和子文件夹。 写入-允许写入文件或将文件和子文件夹添加到文件夹。 列出文件夹内容-此权限只能分配给文件夹。它允许查看和列出文件和子文件夹，以及执行在该文件夹中找到的文件。 读取和执行-允许读取和访问文件内容及其执行。在处理文件夹时，它允许查看和列出文件和子文件夹，以及执行文件。 修改-处理文件时，允许读取、写入和删除文件。在处理文件夹时，它允许读取和写入文件和子文件夹，以及删除文件夹。 完全控制-允许读取、写入、更改和删除任何文件和子文件夹。

通常，文件继承放置它们的文件夹的权限，但用户也可以定义仅分配给特定文件的特定权限。为了简化您的计算，最好仅在文件夹级别编辑权限。

为什么权限对Windows中的共享很重要？

权限很重要，因为当您在Windows中共享某些内容时，实际上是将一组权限分配给特定的用户帐户或用户组。共享文件夹只能由拥有有权访问该文件夹的用户帐户的用户访问。

例如，使用共享向导时，您可以选择用户名或用户组，然后选择以下两个权限级别之一：

读/写-相当于“修改”权限级别。 Read-它相当于“Read&Execute”权限级别。

使用共享向导时，您还将看到名为“所有者”的权限级别。这本身不是权限级别。它只是表示您将要共享的文件夹属于您看到此条目的用户帐户。所有者可以完全控制该文件夹。在第6课中，您将了解有关共享向导以及如何使用它的更多信息。

使用高级共享时，您可以分配以下三个权限级别之一：

完全控制-它允许读取、写入、更改和删除任何文件和子文件夹。 更改-相当于修改权限级别。 读取-它等同于读取和执行权限级别。

在与网络共享资源时，您会遇到一个名为“Everyone”的特殊群体。此用户组代表与网络共享资源的计算机上有或没有用户帐户的任何人。您将在以后的课程中了解到，当您的网络具有非常不同的设备和操作系统时，此用户组非常有用。高级共享将在第7课中详细说明。

为什么在您的网络中使用Microsoft帐户很有用？

使用Microsoft帐户既有好处(例如，可以跨多个设备同步所有应用程序和设置)，也有缺点(例如，您将向Microsoft提供更多数据)。从网络共享的角度来看，如果您的网络包含许多装有Windows 8.x的PC和设备，则使用Microsoft帐户可能会很有用：

您可以使用相同的凭据在所有设备上使用相同的Microsoft帐户登录。 您不必在每台装有Windows 8.x的计算机或设备上创建单独的本地帐户。 共享时设置权限更容易，因为您不必处理多个本地用户帐户。 访问网络共享也更容易，因为您可以在任何地方使用相同的用户帐户登录，并且可以快速访问与其共享的所有内容。

如果你有一个非常多样化的网络，其中包括Mac、Chromebook或Linux PC以及Windows，那么从网络共享的角度来看，使用微软账户不会提供任何特殊的好处。
