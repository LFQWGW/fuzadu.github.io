如何使用Hyper-V创建和运行虚拟机

Hyper-V是Windows内置的虚拟机功能。Hyper-V最初是Windows Server 2008的一部分，但在Windows 8中实现了向台式机的飞跃。Hyper-V允许您在没有任何额外软件的情况下创建虚拟机。

此功能在Windows 7上不可用，它需要Windows 8、8.1或10的专业版或企业版。它还需要支持硬件虚拟化的CPU，如Intel VT或AMD-V，这些功能在大多数现代CPU中都有。

安装Hyper-V

默认情况下，Windows 8或10专业版和企业版系统上不会安装Hyper-V，因此您必须先安装Hyper-V，然后才能使用。值得庆幸的是，您不需要Windows光盘来安装它-您只需单击几个复选框即可。

点击Windows键，键入“Windows Feature”执行搜索，然后单击“打开或关闭Windows功能”快捷方式。选中列表中的Hyper-V复选框，然后单击确定进行安装。出现提示时重新启动计算机。

打开Hyper-V管理器

要实际使用Hyper-V，您需要启动Hyper-V管理器应用程序。您可以在您的已安装程序列表中找到它，您也可以通过搜索Hyper-V来启动它。

Hyper-V管理器应用程序指的是“虚拟化服务器”，它放弃了其作为服务器工具的传统。它可用于在您自己的计算机上运行虚拟机-在这种情况下，您的本地计算机充当本地虚拟化服务器。

设置网络

在Hyper-V管理器中单击本地计算机的名称以查找当前计算机的选项。

您可能希望让虚拟机访问Internet和本地网络，因此需要创建一个虚拟交换机。首先单击Virtual Switch Manager(虚拟交换机管理器)链接。

在列表中选择External以授予虚拟机访问外部网络的权限，然后单击Create Virtual Switch。

之后为虚拟交换机命名，然后单击OK。虽然您应该确保外部网络连接正确，但这里的默认选项应该没问题。确保选择实际连接到Internet的网络适配器，无论是Wi-Fi还是有线以太网。

创建虚拟机

在操作窗格中单击“New”>“Virtual Machine”以创建新的虚拟机。

将出现“New Virtual Machine Wizard”(新建虚拟机向导)窗口。使用这些选项命名您的虚拟机并配置其基本硬件。如果您以前使用过另一个虚拟机程序，那么这一切都应该是不言而喻的。当您到达Configure Networking窗格时，您将需要选择您之前配置的虚拟交换机-如果您没有配置虚拟交换机，您将在此处看到的唯一选项是“未连接”，这意味着您的虚拟机将不会连接到网络，除非您稍后将网络适配器添加到其虚拟硬件。

如果您有包含来宾操作系统安装文件的ISO文件，则可以在该过程结束时选择该文件。Hyper-V会将ISO文件插入虚拟机的虚拟磁盘驱动器中，这样您就可以在之后启动它，并立即开始安装您选择的来宾操作系统。

启动虚拟机

您的新虚拟机将显示在Hyper-V管理器列表中。选择它并“启动”它-单击侧边栏中的Start，单击Action>Start，或者右键单击它并选择Start。虚拟机将启动。

接下来，右键单击虚拟机，然后单击连接以连接到它。然后，您的虚拟机将在桌面的窗口中打开-如果您没有连接到它，它只是在后台运行，没有可见的界面。同样，很容易看出此管理界面是如何为服务器设计的。

连接后，您将看到一个标准虚拟机窗口，其中包含可用于控制虚拟机的选项。如果您曾经使用过VirtualBox或VMware Player，它应该看起来很熟悉。通过正常安装过程在虚拟机中安装来宾操作系统。

安装完操作系统后，单击“操作”>“插入Integration Services安装盘”。打开Windows文件管理器并从虚拟磁盘安装集成服务。这是与VirtualBox来宾添加和VMware工具对应的Hyper-V

使用Hyper-V

使用完虚拟机后，请确保已将其关闭或在Hyper-V管理器窗口中将其关闭-仅关闭窗口并不会真正关闭虚拟机，因此它将在后台继续运行。如果您不希望虚拟机运行，则虚拟机的状态应为“关闭”。

每个虚拟机都有一个设置窗口，您可以使用它来配置其虚拟硬件和其他设置。右键单击虚拟机并选择设置以调整这些选项。其中许多设置只能在虚拟机关闭时修改。

这个工具是由微软创建的，但这并不意味着它只适用于Windows。Hyper-V还可用于运行基于Linux的虚拟机。我们可以在Windows8.1上运行带有Hyper-V的Ubuntu 14.04-不需要特殊配置。

Hyper-V还有其他有用的功能。例如，检查点的工作方式类似于VirtualBox或VMware中的快照。您可以创建检查点，然后稍后将来宾操作系统的状态恢复到该状态。这是一个有用的功能，用于试验可能会在客户操作系统中导致问题的软件或调整。
