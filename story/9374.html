如何写入Mac上的NTFS驱动器

苹果的MacOS操作系统可以读取Windows格式的NTFS驱动器，但不能对其进行开箱即用的写入。以下是获得对NTFS驱动器的完全读/写访问权限的几个解决方案。

如果要写入Mac上的Boot Camp分区，这可能很有用，因为Windows系统分区必须使用NTFS文件系统。但是，对于外部驱动器，您可能应该使用exFAT。MacOS可以本地读写exFAT驱动器，就像Windows一样。

这种方法是免费的，但是需要大量的工作，而且安全性较低。要让你的Mac在读写模式下自动挂载NTFS分区，你必须暂时禁用系统完整性保护，并用更容易受到攻击的二进制文件替换苹果的一个内置工具，所以这种方法存在安全风险。

但是，如果您不介意使用终端，您可以使用FUSE以读写模式手动挂载NTFS分区。这项工作更安全，但工作量更大。

首先，下载MacOS版的Fuse并安装。安装时使用默认选项。

你还需要安装苹果的高级命令行开发工具才能继续。如果您尚未安装它们，可以从Finder>Applications>Utilities打开终端窗口，然后运行以下命令进行安装：

xcode-select --install

当系统提示您安装工具时，单击“安装”。

此外，如果您尚未在Mac上安装，则需要先下载并安装Homebrew。*Homebrew是Mac OS X的“软件包管理器”。*将以下命令复制粘贴到终端窗口中，然后按Enter键进行安装：

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

按Enter键并在系统提示时提供您的密码。该脚本将自动下载并安装HomeBREW。

安装开发人员工具和HomeBREW后，在终端窗口中运行以下命令以安装NTFS-3G：

brew install ntfs-3g

您现在可以在读/写模式下手动挂载NTFS分区。*在终端窗口中，运行以下命令在/Volumes/NTFS处创建挂载点。您只需要这样做一次。

sudo mkdir /Volumes/NTFS

将NTFS驱动器连接到计算机时，运行以下命令以列出所有磁盘分区：

diskutil list

然后，您可以标识NTFS分区的设备名称。只需查找包含WINDOWS_NTFS文件系统的分区。在下面的截图中，是/dev/disk3s1。

NTFS分区可能是由您的Mac自动挂载的，所以您需要先卸载它。运行以下命令，将/dev/disk2s1替换为您的NTFS分区的设备名称。

sudo umount /dev/disk2s1

要挂载驱动器，请运行以下命令，将/dev/disk2s1替换为NTFS分区的主设备名称。

sudo /usr/local/bin/ntfs-3g /dev/disk2s1 /Volumes/NTFS -olocal -oallow_other

您将看到文件系统挂载在/volumes/ntfs上。它还将作为普通挂载驱动器显示在您的桌面上。当您想要拔下插头时，可以正常弹出。

如果您喜欢按照上面的说明手动挂载分区，那么您不必继续。

如果要让Mac自动挂载以读写模式连接的NTFS驱动器，则需要禁用系统完整性保护。

警告：您可能不想这样做！在软件的官方说明中警告这是一个安全风险。您将使用NTFS-3G工具替换Mac中的NTFS挂载工具，NTFS-3G工具将以root用户身份运行。由于Homebrew安装软件的方式，在Mac上运行的恶意软件可能会覆盖这些工具。这可能不是冒险，但是如果你想冒这个险，我们会解释怎么做。

重新启动Mac并在启动时按住Command+R。它将在环境中引导进入特殊的恢复模式。

在恢复模式下从Utilities(实用程序)菜单启动终端，然后运行以下命令：

csrutil disable

完成后，请正常重新启动您的Mac。

在Mac桌面上，再次打开终端窗口并运行以下命令以使NTFS-3G正常运行：

sudo mv /sbin/mount_ntfs /sbin/mount_ntfs.original sudo ln -s /usr/local/sbin/mount_ntfs /sbin/mount_ntfs

最后，重新启用系统完整性保护。重新启动Mac并在启动时按住Command+R进入恢复模式。在恢复模式下启动终端并运行以下命令：

csrutil enable

完成后，重新启动您的Mac。NTFS-WRITE支持现在应该正在运行。

要撤消更改并卸载所有内容，您需要首先禁用系统完整性保护。执行此操作后，请运行以下命令：

sudo rm /sbin/mount_ntfs sudo mv /sbin/mount_ntfs.original /sbin/mount_ntfs brew uninstall ntfs-3g

然后，您可以从系统首选项窗口中的面板中卸载用于MacOS的FUSE，然后重新启用系统完整性保护。

苹果的实验性NTFS-编写支持：不要这样做，认真的

我们不推荐使用下面的方法，因为它是测试最少的。这可能无法正常工作，所以如果您遇到问题，不要责怪我们或苹果。从MacOS 10.12 Sierra开始，它仍然不稳定，而且可能永远不会完全稳定。这实际上只是出于教育目的。

首先，请确保您的驱动器具有方便的单字标签。如果没有，就换个标签。这将使这一过程变得更容易。

您首先需要启动一个终端。导航至Finder>Applications>Utilities>Terminal或按Command+Space，键入Terminal，然后按Enter键。

在终端中键入以下命令，以打开/etc/fstab文件，以便在Nano文本编辑器中进行编辑：

sudo nano /etc/fstab

将以下行添加到Nano，并将“name”替换为您的NTFS驱动器的标签：

LABEL=NAME none ntfs rw,auto,nobrowse

完成后按Ctrl+O保存文件，然后按Ctrl+X关闭Nano。

(如果您有多个要写入的NTFS驱动器，请为每个驱动器添加不同的行。)

将驱动器连接到计算机-拔出它，如果已经连接，则重新连接-您将在“/Volumes”目录下看到它。在Finder窗口中，您可以单击“Go”>“Go to Folder”，然后在框中键入“/Volumes”以访问它。它不会像驱动器通常那样自动弹出并显示在您的桌面上。

要在以后撤消此更改，只需重复上述过程，在Nano中打开/etc/fstab文件。删除添加到文件中的行并保存更改。
