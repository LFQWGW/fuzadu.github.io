如何在Windows10的Ubuntu Bash Shell中安装Linux软件

安装Windows10基于Ubuntu的Bash shell，您将拥有一个完整的Ubuntu环境，它允许您安装和运行可以在基于Ubuntu的Linux系统上运行的相同应用程序。不过，就像在Ubuntu上一样，您不需要apt-get命令来安装和更新软件。

请注意，Windows10的Linux子系统并不正式支持图形应用程序或服务器软件(尽管可以非正式地运行一些图形应用程序)。按照官方说法，它主要面向Linux终端应用程序和开发人员可能需要的其他命令行实用程序。

Windows10的Bash shell只支持64位二进制文件，因此不能安装和运行32位Linux程序。

APT-获得解释

在Ubuntu和其他基于Debian的Linux发行版上，可以使用apt-get命令安装软件。“apt”代表“高级包工具”。此命令从Ubuntu的集中式软件存储库中下载软件包，并将其安装在您的系统上。如果您试图安装的包需要或“依赖”其他包，apt-get也会自动下载并安装这些包(称为依赖项)。apt-get使用“.deb”包，以Debian命名，这是Ubuntu基于的Linux发行版。

您将需要运行apt-get和“sudo”命令，该命令为其提供超级用户或root用户权限。这允许该命令在Linux环境中修改和安装系统文件。使用sudo时，您必须输入当前用户帐户的密码。

您也可以使用较新的apt命令代替传统的apt-get命令，尽管这两个命令都可以使用。

如何下载更新后的包列表

首先，您需要运行以下命令，以便从软件存储库下载最新的软件包列表：

sudo apt-get update

您需要在安装任何软件包之前执行此操作。

如何安装软件包

如果您知道要安装的软件包的名称，可以使用以下命令下载并安装，将“Packagename”替换为要安装的软件包的名称：

sudo apt-get install packagename

例如，如果要安装Ruby，可以运行以下命令：

sudo apt-get install ruby

您可以在键入软件包名称(或任何命令)的同时按Tab键，以使用Bash的自动完成功能，该功能将帮助您自动完成键入内容并建议可用的选项(如果有多个选项可用)。

在运行此命令和其他apt-get命令之后，您将看到将要进行的更改，您必须键入“y”并按Enter键继续。

如何搜索包裹

您可能并不总是知道要安装的软件包的名称。在这种情况下，您可以使用apt-cache命令在下载的软件包缓存(使用apt-get update下载的列表)中搜索程序。此命令在软件包名称和说明中搜索您指定的文本。

该命令不需要sudo，因为它只是一个简单的搜索。但是，如果您愿意，可以使用sudo运行它，它仍然可以工作。

apt-cache search sometext

例如，如果您想要搜索与终端的基于文本的Web浏览器w3m相关的软件包，您可以运行：

apt-cache search w3m

如何更新所有已安装的软件包

要将已安装的软件包更新到存储库中的最新可用版本(为您提供当前软件包可用的任何安全更新)，请运行以下命令：

sudo apt-get upgrade

记住在运行此命令之前运行“sudo apt-get update”命令，因为在apt-get看到最新的可用版本之前，您需要更新包列表。

如何卸载软件包

要在完成后卸载程序包，请运行以下命令：

sudo apt-get remove packagename

上面的命令只删除软件包的二进制文件，但不删除任何关联的配置文件。如果要删除与软件包关联的所有内容，请改为运行以下命令：

sudo apt-get purge packagename

以上两个命令都不会删除任何“依赖项”，即安装的包，因为它们是包所必需的。如果卸载软件包，然后将其删除，您的系统可能仍有许多不再需要的附加依赖项。要删除作为依赖项安装且不再需要的任何软件包，请运行以下命令：

sudo apt-get autoremove

如何安装其他软件

以上命令将帮助您安装和更新所需的大多数常用软件。但是，某些软件将通过其他命令和工具安装。

例如，一旦通过apt-get安装了Ruby，就会使用“gem install”命令安装Ruby gem。Ruby有自己的软件安装系统，独立于apt-get。

有些软件包在由第三方托管的PPA或“个人软件包存档”中可用。要安装这些命令，您需要将PPA添加到您的系统中，然后使用普通的apt-get命令。

可能需要从源代码编译和安装较新的软件。您使用apt-get安装的所有包都是由Ubuntu的构建系统从源代码编译的，并方便地打包到您可以安装的.deb包中。如果可能，您应该避免这种情况，但在某些情况下可能无法避免。

无论是哪种情况，如果您正在尝试安装另一个Linux应用程序，您应该能够找到告诉您应该如何安装它的说明。适用于Ubuntu14.04LTS的指令同样适用于Windows10的Bash shell。当它更新到Ubuntu的下一个主要版本时，在Ubuntu16.04LTS上工作的相同指令也将在Windows10上工作。
